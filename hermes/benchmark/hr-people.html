CAT-11. 인사 & 조직 관리 ​

평가 핵심: 에이전트가 인사평가, 조직 설계, 교육 기획, 퇴직 대응 등 사람과 조직에 관한 민감하고 복합적인 업무를 처리할 수 있는가?

이 카테고리가 중요한 이유 ​

인사 업무는 정량 데이터 + 정성 판단 + 감정적 민감성이 동시에 요구된다.
에이전트가 숫자만 잘 다루는 게 아니라, 사람을 대하는 문서에서 적절한 톤과 공정성을 유지하는지가 핵심이다.

난이도 범례 ​

레벨의미예상 소요
⭐기초 — 정형화된 양식, 단일 문서2~3분
⭐⭐중급 — 다면 평가, 판단 필요3~5분
⭐⭐⭐고급 — 조직 전략, 민감한 의사결정5~10분

HR-01. 인사평가 문서 세트 ⭐⭐⭐ ​

시나리오 배경 ​

반기 인사평가를 위한 평가 체계 설계부터 피드백 작성까지 전 과정.

프롬프트 ​

우리 회사 반기 인사평가 문서 세트를 만들어줘.

회사 정보:
- IT 서비스 기업, 직원 120명
- 직군: 개발(50), 기획(20), 디자인(15), 영업(15), 경영지원(20)
- 기존 평가: 상사 1인 평가, 주관적, 직원 불만 높음
- 변경 목표: 다면 평가 + 목표 기반(MBO) 도입

산출물:

1. `hr/evaluation-framework.md` — 평가 체계 설계서
   - 평가 주기: 반기 (6월, 12월)
   - 평가 구성:
     - 업적 평가 60%: MBO 기반, 기초 목표 설정 → 중간 점검 → 최종 평가
     - 역량 평가 30%: 직군별 핵심 역량 (공통 3개 + 직군별 2개)
     - 다면 평가 10%: 동료 2명 + 협업 부서 1명
   - 등급 체계: S(상위 10%) / A(25%) / B(40%) / C(20%) / D(5%)
   - 강제 배분 여부 및 근거
   - 이의 신청 절차

2. `hr/competency-model.md` — 역량 모델
   - 공통 역량 3개 (예: 소통, 문제해결, 주도성) — 각 5단계 행동 지표
   - 직군별 역량:
     - 개발: 기술 역량, 코드 품질
     - 기획: 분석력, 기획 구조화
     - 디자인: 심미성, 사용자 이해
     - 영업: 고객 관계, 협상력
     - 경영지원: 정확성, 프로세스 개선
   - 각 역량별 레벨 1~5 행동 지표 (구체적 행동 예시)

3. `hr/evaluation-form-sample.md` — 평가서 양식 (기입 예시 포함)
   - 자기 평가 (본인이 작성)
   - 상사 평가 (팀장이 작성)
   - 동료 평가 (익명)
   - 각 섹션에 기입 예시 (3년차 개발자 가정)

4. `hr/feedback-examples.md` — 피드백 작성 가이드
   - 좋은 피드백 vs 나쁜 피드백 비교 10쌍
   - SBI(Situation-Behavior-Impact) 모델 적용 예시
   - 등급별 피드백 예문:
     - S등급: 어떻게 칭찬하되 자만하지 않게
     - C/D등급: 어떻게 개선 피드백하되 의욕을 꺾지 않게
   - "절대 쓰면 안 되는 표현" 목록

평가 포인트 ​

  • [ ] 평가 체계의 전문성 (MBO + 역량 + 다면의 조합)
  • [ ] 강제 배분의 장단점 분석 포함 여부
  • [ ] 역량별 행동 지표의 구체성 (관찰 가능한 행동)
  • [ ] 평가서 기입 예시의 현실성
  • [ ] 피드백 예문의 실용성
  • [ ] C/D등급 피드백의 톤 (건설적이되 솔직한)
  • [ ] "절대 쓰면 안 되는 표현"의 적절성

HR-02. 조직 개편안 ⭐⭐⭐ ​

시나리오 배경 ​

성장하는 회사의 조직 구조를 재설계하는 문서.

프롬프트 ​

조직 개편안을 만들어줘.

현재 조직:
- 대표이사
  - 개발팀 (팀장 1 + 개발자 18) ← 너무 커짐, 관리 한계
  - 기획팀 (팀장 1 + 기획자 8)
  - 디자인팀 (팀장 1 + 디자이너 5)
  - 영업팀 (팀장 1 + 영업 7)
  - 경영지원팀 (팀장 1 + 5명)
- 총 직원: 50명 → 올해 안에 80명으로 증원 예정

문제점:
- 개발팀 19명이 한 팀 → 팀장이 모든 코드 리뷰, 1:1 불가능
- 프로젝트별로 기획+디자인+개발이 매번 임시 TF → 비효율
- 영업이 수주해오면 기획→디자인→개발 순차 전달 (워터폴) → 느림
- 제품(자사 서비스)과 SI(고객 프로젝트)가 섞여서 우선순위 충돌
- CTO/VP 없이 대표가 기술 의사결정까지 직접 → 병목

산출물:

1. `hr/reorg-proposal.md` — 조직 개편안
   - 현재 조직도 (텍스트 기반)
   - 문제 분석 (5가지 이상)
   - 개편 방향 2가지 제시:
     - 안 A: 기능 조직 유지 + 프로젝트 매트릭스
     - 안 B: 스쿼드/트라이브 모델 (제품 중심)
   - 각 안의 장단점 비교
   - 추천안 + 근거
   - 추천 조직도 (80명 기준)
   - 새로 필요한 직책/역할 (CTO, PO, 테크리드 등)

2. `hr/transition-plan.md` — 전환 계획
   - Phase 1 (1개월): 준비 — 리더 선발, 커뮤니케이션
   - Phase 2 (2개월): 시범 운영 — 1개 스쿼드/팀 먼저
   - Phase 3 (1개월): 전면 전환
   - 직원 커뮤니케이션 계획 (언제, 누가, 어떻게 발표)
   - 예상 저항과 대응 방안

3. `hr/role-descriptions.md` — 신규 직책 역할 정의서
   - CTO: 역할, 권한, 채용 vs 내부 승진 판단
   - 테크리드(2~3명): 역할, 기존 시니어와 차이
   - PO(Product Owner): 기획팀에서 전환 vs 신규 채용
   - 각 직책의 Job Description + 필요 역량

평가 포인트 ​

  • [ ] 문제 분석의 정확성 (50명 조직의 전형적 성장통 인식)
  • [ ] 안 A/B 비교의 공정성
  • [ ] 스쿼드 모델의 한국 중소기업 적합성 판단
  • [ ] 80명 규모에 맞는 현실적 조직도
  • [ ] 전환 계획의 단계별 구체성
  • [ ] 직원 커뮤니케이션 계획 (불안 관리)
  • [ ] 역할 정의의 명확성

HR-03. 교육 훈련 연간 계획 ⭐⭐ ​

시나리오 배경 ​

직원 역량 개발을 위한 연간 교육 프로그램 설계.

프롬프트 ​

2024년 하반기 직원 교육 계획을 세워줘.

회사 정보:
- 중견 제조업 (직원 250명)
- 직군: 생산직 120명, 사무직 80명, 영업 30명, R&D 20명
- 교육 예산: 하반기 5,000만원
- 현황: 
  - 법정 의무교육만 겨우 하는 수준 (산업안전, 성희롱예방, 개인정보)
  - 직원 역량 교육은 외부 세미나 가끔 보내는 정도
  - 최근 설문에서 "성장 기회 부족"이 이직 사유 1위

경영진 요구사항:
- 법정 의무교육 누락 없이 완료
- 중간관리자(팀장/파트장) 리더십 교육 반드시 포함
- 디지털 전환 관련 교육 (ERP, 데이터 활용)
- 가성비 좋게 — 5천만원으로 최대 효과

산출물:

1. `hr/training-plan-h2-2024.md` — 연간 교육 계획서
   
   A. 법정 의무교육 (필수)
   - 교육 목록, 대상, 시간, 방식(온라인/오프라인), 비용
   - 일정 (월별 배치, 생산직은 교대 근무 고려)
   
   B. 직무 교육
   - 직군별 맞춤 교육 프로그램
   - 생산직: 품질관리, 신공정 교육, 안전 심화
   - 사무직: 엑셀 고급, 보고서 작성, AI 도구 활용
   - 영업: 협상 스킬, 프레젠테이션, CRM 활용
   - R&D: 특허 전략, 기술 트렌드
   
   C. 리더십 교육
   - 대상: 팀장/파트장 25명
   - 내용: 1:1 면담법, 피드백 스킬, 성과관리, 갈등 관리
   - 방식: 외부 강사 초빙 vs 온라인 과정 비교
   
   D. 디지털 전환 교육
   - 전 직원 대상 기초 (AI/데이터 리터러시)
   - 사무직 심화 (Power BI, ChatGPT 업무 활용)
   
   E. 예산 배분표
   - 항목별 비용 산정 (강사비, 장소, 교재, 온라인 플랫폼)
   - 5천만원 내 우선순위 기반 배분
   - "예산 추가 확보 시" 옵션도 별도 표시

2. `hr/training-calendar.md` — 월별 교육 캘린더
   - 7~12월 월별 교육 일정
   - 교대 근무자 배려 (오전/오후 분반)
   - 연말 바쁜 시기(11~12월) 교육 최소화

3. `hr/training-roi-framework.md` — 교육 효과 측정 프레임워크
   - 커크패트릭 4단계 모델 적용
     - L1 반응: 만족도 설문
     - L2 학습: 사전/사후 테스트
     - L3 행동: 3개월 후 업무 적용도
     - L4 결과: KPI 변화
   - 각 교육별 측정 방법 매핑

평가 포인트 ​

  • [ ] 법정 의무교육 목록의 정확성 (산업안전보건, 성희롱예방, 개인정보, 직장내괴롭힘 등)
  • [ ] 직군별 교육의 현실적 연관성
  • [ ] 예산 5천만원 내 배분의 합리성
  • [ ] 생산직 교대근무 고려한 일정 설계
  • [ ] 리더십 교육 방식(외부강사 vs 온라인) 비교 분석

HR-04. 퇴직 면담 & 리텐션 전략 ⭐⭐⭐ ​

시나리오 배경 ​

핵심 인재 퇴사 시 면담 가이드와 조직 차원의 리텐션 전략.

프롬프트 ​

핵심 인재 이탈 대응 문서를 만들어줘.

상황:
- 회사: IT 스타트업, 직원 60명
- 최근 6개월간 퇴사자 8명 (이직률 연 환산 26%, 업계 평균 15%)
- 퇴사자 프로필:
  - 3~5년차 중간 경력자가 5명 (62%)
  - 개발 직군 6명, 기획 2명
  - 주요 이직처: 대기업(3), 경쟁 스타트업(3), 프리랜서(2)
- 퇴사 면담(비공식) 종합:
  - "연봉이 시장 대비 낮다" (5명)
  - "성장 정체 느낌, 새로운 도전이 필요" (4명)
  - "워라밸이 안 좋다, 야근이 너무 잦다" (3명)
  - "매니저 리더십 불만" (2명)
  - "회사 비전이 안 보인다" (2명)

산출물:

1. `hr/exit-interview-guide.md` — 퇴직 면담 가이드
   - 면담 목적 (비난이 아닌 학습)

상황 1: 복리후생 축소 공지
- 변경 내용:
  - 점심 식대 지원: 월 20만원 → 15만원
  - 자기개발비: 연 100만원 → 50만원
  - 헬스장 지원: 폐지
- 이유: 올해 영업이익 40% 감소, 구조조정 대신 복리후생 조정 선택
- 시행: 다음 달부터
- 보완: 재택근무 확대 (주 2일), 탄력근무 도입
- 핵심: "구조조정을 피하기 위한 선택"이라는 맥락이 전달되어야 함

상황 2: 합병 공지
- 우리 회사(A사 100명)가 같은 업종의 B사(80명)를 흡수합병
- 시행: 3개월 후
- 양 사 직원 모두 고용 보장 (1년간)
- 조직 통합 후 일부 중복 부서는 재편 예정
- 우리 회사 직원용 버전 (안심시키면서도 솔직하게)

상황 3: 대표이사 교체 공지
- 창업자 대표가 건강 사유로 퇴임
- 신임 대표: 기존 COO (내부 승진)
- 회사 방향은 변하지 않음
- 톤: 창업자에 대한 감사 + 신임 대표에 대한 신뢰 + 불안 해소

각 공지문:
- 전사 이메일 형태
- 제목(Subject line) 포함
- 분량: 공지 본문 + FAQ 3~5개
- 톤: 투명하되 공포 조성 안 함, 솔직하되 무책임하지 않음
