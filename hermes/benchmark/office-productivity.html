CAT-10. 업무 자동화 & 생산성 ​

평가 핵심: 에이전트가 엑셀 수식, 데이터 정리, 양식 제작, 업무 매뉴얼 등 일상적인 오피스 업무를 자동화할 수 있는가?

이 카테고리가 중요한 이유 ​

"HTML 파일 만들어봐"는 개발자만의 테스트다.
현실에서 직장인이 에이전트에게 맡기고 싶은 건 엑셀 정리, 양식 만들기, 매뉴얼 작성 같은 일이다.
이 카테고리는 비개발자 직장인의 실제 업무 패턴을 반영한다.

난이도 범례 ​

레벨의미예상 소요
⭐기초 — 단순 변환, 정해진 형식2~3분
⭐⭐중급 — 로직 설계, 복수 산출물3~5분
⭐⭐⭐고급 — 복잡한 자동화, 종합 산출물5~10분

OP-01. 엑셀 수식 & 데이터 처리 ⭐⭐ ​

시나리오 배경 ​

직장인이 가장 많이 요청하는 유형 — "이 엑셀 좀 정리해줘".

프롬프트 ​

아래 엑셀 업무를 해결하는 Python 스크립트를 만들어줘. 
(openpyxl 사용)

테스트용 `sales_raw.xlsx`를 먼저 만들어 (시트 3개):

Sheet1 "원본데이터":
- 1000행의 판매 데이터
- 컬럼: 날짜, 영업사원, 지역, 고객명, 제품코드, 수량, 단가, 할인율
- 더러운 데이터 포함:
  - 날짜: "2024-01-15", "1/15/2024", "24.1.15" 혼재
  - 영업사원: "김영수", "김 영수", " 김영수 " (공백 문제)
  - 수량에 가끔 음수 (반품)
  - 할인율: 0.1, 10%, "10퍼센트" 혼재
  - 빈 행 20개 랜덤 삽입

Sheet2 "제품마스터":
- 제품코드, 제품명, 카테고리, 표준단가
- 50개 제품

Sheet3 "목표":
- 영업사원별 월별 목표 매출액

처리 스크립트 `process_sales.py`:
1. 데이터 정제
   - 날짜 통일 (YYYY-MM-DD)
   - 이름 공백 정리
   - 할인율 숫자 통일 (0~1 범위)
   - 빈 행 제거
   
2. `report.xlsx` 생성 (시트 4개):

   Sheet "월별요약":
   - 월별 매출액 (수량×단가×(1-할인율))
   - 전월대비 증감률
   - 셀 서식: 천단위 콤마, 증가는 파랑, 감소는 빨강

   Sheet "영업사원별":
   - 사원별 총 매출, 건수, 평균 단가
   - 목표 대비 달성률 (Sheet3 조인)
   - 달성률 100% 이상 → 초록 배경
   - 달성률 80% 미만 → 빨강 배경

   Sheet "제품분석":
   - 카테고리별 매출 비중
   - 제품별 판매량 TOP 10
   - 반품률 (음수 수량 / 전체 수량)

   Sheet "피벗":
   - 지역 × 카테고리 크로스탭 (매출 합계)
   - 행 합계, 열 합계 포함
   - 최대값 셀 볼드 처리

각 시트에:
- 헤더 행: 파랑 배경 + 흰색 글자 + 볼드
- 열 너비 자동 조정
- 필터 자동 설정
- 숫자: 천단위 콤마
- 금액: ₩ 표시

평가 포인트 ​

  • [ ] 더러운 데이터 생성의 현실성
  • [ ] 날짜 파싱 3가지 형식 대응
  • [ ] 할인율 정규화의 정확성
  • [ ] 조건부 서식 (증감 색상, 달성률 배경색)
  • [ ] 피벗 테이블의 정확한 계산
  • [ ] 열 너비 자동 조정 + 필터 설정
  • [ ] 실제 엑셀 파일로 열었을 때 서식 정상 여부

OP-02. 업무 프로세스 매뉴얼 ⭐⭐ ​

시나리오 배경 ​

신입 사원이 보고 따라할 수 있는 업무 매뉴얼 작성.

프롬프트 ​

아래 업무의 상세 매뉴얼을 만들어줘.

업무: 월말 정산 및 세금계산서 발행

대상 독자: 경리 업무 처음 맡은 신입사원
전제: ERP 시스템 "위하고" 사용, 법인카드 사용

매뉴얼 포함 내용:

1. 업무 개요
   - 전체 흐름도 (텍스트 기반)
   - 월별 일정 (매월 몇 일까지 뭘 해야 하는지)
   - 관련 부서 및 담당자

2. 일일 업무
   - 법인카드 사용 내역 확인 (ERP 화면 경로 포함)
   - 영수증 대조 및 증빙 처리
   - 미확인 건 담당자에게 확인 요청 (이메일 템플릿 포함)

3. 월말 정산 (Step-by-Step)
   - Step 1: 미처리 건 마감 (체크리스트)
   - Step 2: 부서별 경비 집계
   - Step 3: 예산 대비 실적 비교표 작성
   - Step 4: 이상 항목 확인 (어떤 게 이상인지 기준 포함)
   - Step 5: 팀장 결재 요청

4. 세금계산서 발행
   - 발행 대상 확인 방법
   - ERP에서 발행 절차 (가상의 화면 경로)
   - 수정 세금계산서 발행이 필요한 경우
   - 마감일: 다음 달 10일까지 (놓치면 가산세!)
   - 국세청 전자세금계산서 시스템 연동 확인

5. 자주 하는 실수 & 대처법
   - 금액 불일치 시
   - 세금계산서 발행 누락 시
   - 카드 사용 부인 건 처리

6. 참고 자료
   - 관련 법규 (부가가치세법 관련 조항)
   - 사내 결재 양식 다운로드 경로
   - 문의처 (세무사, ERP 고객센터)

파일: `manuals/monthly-closing-manual.md`

주의: "ERP에서 세금계산서를 발행한다" 수준이 아니라
"왼쪽 메뉴 > 회계관리 > 매출전표 > 세금계산서 발행 버튼 클릭" 수준으로 구체적으로.
물론 실제 ERP 화면은 모르니까 가상이지만, 그 수준의 상세도를 보여줘.

평가 포인트 ​

  • [ ] 전체 흐름도의 명확성
  • [ ] 월별 일정의 구체성 (일자 포함)
  • [ ] Step-by-Step의 상세도 (화면 경로 수준)
  • [ ] 이메일 템플릿의 실용성
  • [ ] "자주 하는 실수"의 현실성
  • [ ] 가산세 등 법적 주의사항 포함
  • [ ] 신입사원이 혼자 따라할 수 있는 수준인지

OP-03. 설문조사 설계 & 분석 ⭐⭐ ​

시나리오 배경 ​

직원 만족도, 고객 조사 등 설문을 설계하고 결과를 분석하는 능력.

프롬프트 ​

직원 만족도 설문조사를 설계하고, 가상 결과를 분석해줘.

배경:
- 회사: IT 기업, 직원 150명
- 목적: 반기별 직원 만족도 조사 (이번이 3회차)
- 이전 조사: 전체 만족도 3.2/5 (1회차), 3.5/5 (2회차)
- 최근 이슈: 하이브리드 근무 도입, 평가 제도 변경, 구내식당 리뉴얼

산출물:

1. `survey/employee-satisfaction-survey.md` — 설문지
   - 총 25~30문항
   - 영역: 업무환경(5), 동료/팀(5), 리더십(5), 보상(5), 
     성장/개발(5), 하이브리드근무(3~5)
   - 문항 유형: 5점 리커트 척도 20개 + 객관식 5개 + 주관식 3개
   - 설문 소개문, 익명성 안내, 소요시간 안내 포함
   - 각 문항에 "이 문항으로 무엇을 측정하는가" 설계 의도 주석

2. `survey/mock-results.json` — 가상 응답 데이터
   - 응답자 120명 (응답률 80%)
   - 부서별, 연차별 분포 반영
   - 현실적 응답 패턴:
     - 대체로 정규분포이지만 보상 영역은 낮게 편향
     - 하이브리드 근무 만족도는 부서별 편차 큼
     - 주관식: 긍정 40%, 개선요청 45%, 불만 15%

3. `survey/analysis-report.md` — 분석 보고서
   - 전체 만족도 (이전 차수 대비 트렌드)
   - 영역별 점수 & 전회 대비 변화
   - 부서별 비교 (편차가 큰 부서 하이라이트)
   - 연차별 비교 (주니어 vs 시니어)
   - 주관식 응답 키워드 분석 (빈도수 기반)
   - 핵심 인사이트 3개
   - 개선 제안 (우선순위 포함)
   - 경영진 보고용 1페이지 요약

분석 스크립트: `survey/analyze.py` (JSON 읽어서 분석)

평가 포인트 ​

  • [ ] 설문 문항의 전문성 (리커트 척도 적절한 사용)
  • [ ] 설계 의도 주석의 유용성
  • [ ] 가상 데이터의 현실적 분포
  • [ ] 트렌드 분석 (3회차 데이터 비교)
  • [ ] 부서별/연차별 교차 분석
  • [ ] 주관식 키워드 분석 구현
  • [ ] 핵심 인사이트의 actionable 여부
  • [ ] 경영진 1페이지 요약의 밀도

OP-04. 계약서 & 법률 문서 검토 보조 ⭐⭐⭐ ​

시나리오 배경 ​

법률 문서를 읽고 핵심 조항을 추출·요약·리스크 식별하는 능력.

프롬프트 ​

아래 계약서를 검토하고 리스크 분석을 해줘.

먼저 `contract/software-license-agreement.md`에 가상의 SW 라이센스 계약서를 작성해:
- 계약 당사자: 우리 회사(라이센시) vs 글로벌 SaaS 업체(라이센서)
- 계약 기간: 3년
- 라이센스 비용: 연 1.2억원 (사용자 200명 기준)
- 주요 조항 20개 이상 포함:
  - 사용 범위, 사용자 수 제한, 서브라이센스 금지
  - SLA (가용성 99.5%, 미달성 시 크레딧)
  - 데이터 소유권 및 처리
  - 보안 의무 (ISO 27001 인증)
  - 계약 해지 조건 (30일 전 서면 통보)
  - 자동 갱신 조항 (해지 통보 없으면 1년 자동 연장)
  - 가격 인상 조항 (연 최대 10% 인상 가능)
  - 배상 한도 (연간 라이센스 비용의 100%)
  - 준거법 (미국 캘리포니아주)
  - 분쟁 해결 (중재, ICC 규칙)
- 의도적으로 라이센시(우리)에게 불리한 조항 5개 이상 숨겨둬

검토 결과:
`contract/contract-review.md`에 작성:

1. 계약 요약 (1페이지)
   - 핵심 조건 테이블 (비용, 기간, SLA, 해지 등)
   
2. 리스크 분석 (조항별)
   - 🔴 고위험 (즉시 협상 필요)
   - 🟡 중위험 (주의 필요)
   - 🟢 저위험/수용 가능
   - 각 리스크별: 무엇이 문제인지, 왜 문제인지, 수정 제안

3. 협상 포인트 TOP 5
   - 우선순위대로
   - 각각: 현재 조항 → 우리가 원하는 조항 → 타협안

4. 비교 체크리스트
   - 업계 표준 대비 이 계약이 유리/불리한 점

주의: 법률 자문을 대체하는 것이 아님을 명시.
핵심은 비법률가(팀장, 임원)가 읽고 협상 포인트를 파악하는 것.

평가 포인트 ​

  • [ ] 계약서의 현실성 (실제 SW 라이센스 조항 반영)
  • [ ] 숨겨진 불리한 조항 식별 정확도
  • [ ] 리스크 등급 분류의 합리성

OP-05. 발표 자료 기획 & 스토리보드 ⭐⭐ ​

시나리오 배경 ​

PPT를 직접 만드는 것이 아닌, 슬라이드 구성과 내용을 기획하는 능력.

프롬프트 ​

아래 발표를 위한 PPT 기획안(스토리보드)을 만들어줘.

발표 정보:
- 주제: 2024년 상반기 사업 실적 보고
- 발표자: 대표이사
- 청중: 전 직원 (150명, 타운홀 미팅)
- 시간: 20분
- 분위기: 실적은 양호하지만, 하반기 도전이 큼 → "잘했지만 방심 금물" 톤

데이터:
- 매출: 250억 (목표 230억, 109% 달성)
- 영업이익: 25억 (목표 20억, 125%)
- 신규 고객: 45사 (목표 40사)
- 이탈 고객: 8사 (전년 5사, 증가 ⚠️)
- 신규 채용: 23명, 퇴사: 12명 (이직률 8%)
- 핵심 성과: A 프로젝트 성공적 론칭, B 파트너십 체결
- 하반기 도전: 경쟁 심화, 핵심 인력 유출 우려, 신규 사업 투자 필요

스토리보드에 포함:

각 슬라이드별:
- 슬라이드 번호, 제목
- 레이아웃 유형 (타이틀, 텍스트+차트, 숫자 강조, 이미지+텍스트 등)
- 핵심 메시지 (이 슬라이드에서 전달할 한 문장)
- 들어갈 내용 (텍스트, 데이터, 차트 종류)
- 디자인 노트 (색상, 강조, 주의사항)
- 발표자 노트 (30초~1분, 실제로 말할 내용)

구성:
- 총 15~18장
- 오프닝: 에너지 있게 시작
- 실적 파트: 좋은 소식 먼저 (긍정 모멘텀)
- 주의 파트: 이탈 고객 증가, 인력 이슈 (솔직하되 해결 의지)
- 하반기 방향: 구체적 전략 3가지
- 클로징: 동기부여 메시지

파일: `presentations/h1-2024-business-review-storyboard.md`

  • [ ] 슬라이드 구성의 스토리 흐름 (기승전결)
  • [ ] 레이아웃 유형의 적절한 배분 (숫자 강조 vs 차트 vs 텍스트)
  • [ ] "잘했지만 방심 금물" 톤의 일관된 유지

OP-08. 데이터 기반 의사결정 지원 ⭐⭐⭐ ​

시나리오 배경 ​
