왜 지금 Hermes가 뜨는가 ​

Trend Analysis2026 Q253.1k Stars in 6주

2026년 2월 공개된 Hermes Agent는 6주 만에 53,000개 이상의 GitHub 스타, 375명의 기여자, 7개 메이저 릴리스를 기록하며 오픈소스 AI 에이전트 생태계의 이목을 한 몸에 받고 있습니다. 이 페이지는 그 인기의 구조적 원인을 단순한 기능 나열이 아니라 시장·브랜드·타이밍·기술적 선택 네 축에서 분석합니다.

원인 1 · Nous Research 브랜드 후광 ​

Hermes Agent가 완전히 새로운 조직에서 나왔다면 이 속도는 불가능했습니다. Nous Research는 이미 Hermes LLM 시리즈로 오픈소스 LLM 파인튜닝 커뮤니티에서 확고한 신뢰를 쌓아둔 상태였습니다.

기존 자산 ​

Nous Hermes 모델 시리즈
Llama·Mistral 기반 파인튜닝으로 2023년부터 Hugging Face 다운로드 상위권
DPO·DSPO 연구
정렬·학습 기법 논문으로 연구자 커뮤니티 접점
Nous Portal
자체 모델 API 서비스, 이미 400+ 모델 서빙 인프라
Discord 커뮤니티
수만 명의 활성 연구자·개발자
WorldSim·여타 프로토타입
창의적 에이전트 실험으로 "재미있는 것을 만드는 팀"이라는 이미지

새 에이전트를 출시했을 때 "또 하나의 LangChain 클론"으로 의심받지 않고 커뮤니티가 즉시 "이 사람들이 만들면 뭐가 달라도 다르겠지"라는 기대로 반응했습니다. 릴리스 첫 주에 수천 스타가 쌓인 건 이 누적 신뢰의 결과입니다.

브랜드 일관성 ​

Hermes 이름을 모델→에이전트로 일관되게 확장한 것도 전략적입니다. 기존 Hermes 모델 사용자는 자연스럽게 "Hermes Agent가 Hermes 모델을 쓰면 최적화되어 있겠지" 기대했고, 실제로 Nous Portal이 400+ 모델을 무료/저가로 제공하면서 이 기대를 충족시켰습니다(v0.5.0·v0.8.0).

원인 2 · "자기개선"이라는 선명한 포지셔닝 ​

기존 에이전트 프레임워크가 기능 나열로 자신을 설명했다면, Hermes는 한 문장으로 자신을 규정했습니다.

"The only agent with a built-in learning loop — it creates skills from experience, improves them during use, nudges itself to persist knowledge."

이 문장은 경쟁자를 자동으로 제외합니다:

이 문장이 내포하는 것

  • 쓸수록 더 좋아진다
  • 반복 작업이 자동화된다
  • 사용자별로 개인화된다
  • 팀 지식이 축적된다
  • 장기 투자 가치가 있다

이 문장이 말하는 경쟁자

  • Claude Code — 좋지만 학습 안 됨
  • Codex — 쓸 때마다 처음부터
  • AutoGPT — 무한 루프에 빠짐
  • CrewAI — 역할만 부여, 개선 없음
  • LangGraph — 그래프는 있으나 경험 축적 없음

마케팅에서 가장 강력한 메시지는 **"무엇이 아닌가"**를 암묵적으로 말하는 것입니다. 학습 루프는 기술적 차별점일 뿐만 아니라 메시징의 차별점이었습니다.

원인 3 · 2026년 초의 시장 공백 ​

2025년 말부터 2026년 초는 AI 에이전트 시장이 **"약속은 많고 실속은 적다"**는 피로감에 빠진 시기였습니다.

2023 · AutoGPT 열풍       →  무한 루프, 비용 폭주, 결과 불안정
2024 · 에이전트 이론 확산    →  ReAct, CoT, 프롬프트 엔지니어링 포화
2025 · Claude Code / Codex →  IDE 밀착 성공, 하지만 벤더 lock-in
2025 말 · LangGraph / CrewAI →  프레임워크 복잡, 실무 격차
───────────────────────────────────
2026 Q1 · 요구사항 정리
  ✓ 실제 도구 실행 (마크다운 에이전트 말고)
  ✓ 오픈소스, 자체 호스팅 가능
  ✓ 모델 중립 (API 키 자유)
  ✓ 쓸수록 개선 (매번 처음부터 가르치기 피로)
  ✓ 프로덕션 수준 보안
  ✓ 15+ 메시징 플랫폼 (어디서든 접근)

이 다섯 가지 요구를 동시에 만족하는 프레임워크가 없었습니다. Claude Code는 3·4·5를 놓치고, OpenClaw는 4를 놓치고, AutoGPT는 2·5를 놓치고, 상용 SaaS는 전부를 놓쳤습니다.

Hermes는 이 모든 요구를 한 번에 체크한 채로 등장했습니다. 6주 동안 53k 스타는 "드디어 우리가 기다리던 것"이라는 반응에 가깝습니다.

모델 종속 탈출 ​

Claude Code는 Claude만, Codex는 GPT만 쓸 수 있습니다. 2026년은 Gemini 2.0, Claude 4 Opus, GPT-5, Llama 4 등 매달 새 모델이 나오는 시기입니다. Hermes는 v0.2.0부터 200+ 모델 지원으로 시작해 v0.5.0에서 400+, v0.8.0에서 Gemini 네이티브까지 추가했습니다.

사용자는 **"이번에 나온 모델을 바로 써볼 수 있다"**는 이유만으로 Hermes를 씁니다. 이건 구조적이라 경쟁자가 쉽게 따라잡기 어렵습니다.

원인 5 · 6주 7릴리스의 속도 신호 ​

오픈소스 프로젝트의 건강도는 릴리스 빈도에서 드러납니다.

v0.2.0 (2026-03-12) · 최초 공개, 216 PR, 63 contributors
v0.3.0 (2026-03-17) · 5일 후, 스트리밍 + 플러그인
v0.4.0 (2026-03-23) · 6일 후, OpenAI API 서버 + 6개 플랫폼
v0.5.0 (2026-03-28) · 5일 후, 공급망 하드닝 + HF
v0.6.0 (2026-03-30) · 2일 후, 프로파일 + MCP 서버 모드
v0.7.0 (2026-04-03) · 4일 후, 복원력 + 플러그블 메모리
v0.8.0 (2026-04-08) · 5일 후, 백그라운드 알림 + Gemini

릴리스 간격의 중앙값이 5일입니다. 이것이 커뮤니티에 보내는 신호:

  1. 이슈가 빠르게 반영된다 — 보고한 버그가 다음 주에 고쳐짐
  2. 프로젝트가 살아있다 — "유지보수 중단" 리스크가 낮음
  3. 방향성이 분명하다 — 릴리스마다 코드네임이 있고 테마가 있음
  4. 기여가 환영받는다 — PR이 빠르게 머지됨

특히 v0.5.0의 litellm 공급망 공격 대응 속도는 전설적이었습니다. 커뮤니티가 공격을 감지한 지 며칠 안에 litellm을 제거하고 해시 핀닝을 도입한 릴리스를 낸 것이 "신뢰할 수 있는 팀"이라는 이미지를 굳혔습니다. 자세한 것은 보안 체계 · 공급망 하드닝 참조.

원인 6 · 스킬 생태계의 네트워크 효과 ​

agentskills.io 허브가 진짜 네트워크 효과를 만들어내는 첫 에이전트 생태계입니다.

어떻게 작동하는가 ​

사용자 A가 스킬 작성 · 배포
        ↓
    agentskills.io에 643+ 스킬 축적
        ↓
사용자 B가 "ECR 배포" 검색 → A의 스킬 발견
        ↓
B가 자기 맥락에 맞게 refine
        ↓
A의 스킬에 refinement 반영 PR
        ↓
원본이 점점 성숙해져 C, D, E에게도 가치
        ↓
[더 많은 사용자 → 더 많은 스킬 → 더 많은 가치]

스킬이 쌓일수록 Hermes를 쓸 이유가 커집니다. 이것이 AutoGPT 같은 초기 프레임워크가 만들지 못한 구조적 moat입니다.

오픈 표준이라는 똑똑한 선택 ​

agentskills.io는 Nous Research가 소유했지만 벤더 중립 오픈 표준으로 설계되었습니다. 이론적으로 다른 에이전트도 같은 스킬을 로드할 수 있습니다. 이 개방성이 역설적으로 Hermes로의 집중을 강화합니다 — "나중에 다른 프레임워크로 옮겨도 내 스킬은 살아남는다"는 심리적 안전감.

원인 7 · 복합 플랫폼 접점의 첫 사례 ​

Hermes 이전의 에이전트는 한 접점만 잘했습니다:

  • Claude Code / Codex / Cursor — IDE만
  • AutoGPT — 웹만
  • CrewAI — API만
  • 각종 Slack 봇 — Slack만

Hermes는 v0.2.0 시점에 이미 6개 접점(Telegram, Discord, Slack, WhatsApp, Signal, Email)을 지원했고, v0.4.0에서 6개 더(DingTalk, SMS, Mattermost, Matrix, Webhook, OpenAI API)를 추가했습니다. v0.6.0에서 Feishu, WeCom까지.

"어디서든 같은 에이전트"의 힘 ​

[Telegram 출근길]  → "어제 배포한 거 롤백해야 할 것 같아"
[CLI 사무실]      → "아까 얘기한 롤백 진행해줘" [자동 컨텍스트 복원]
[Slack 팀 채널]  → "@Hermes 롤백 상태 팀에 공유"
[Email 저녁]      → 완료 리포트 수신

이 시나리오는 한 번 경험하면 돌아갈 수 없습니다. 트위터와 YouTube에서 "Telegram에서 Claude Code를 제어하는 법" 같은 영상이 바이럴이 된 것은 이 경험의 체감 질이 달랐기 때문입니다.

원인 8 · 프로덕션 수준 보안의 신호 ​

2025년 말 LangChain 계열의 보안 사고가 여러 건 있었습니다. 에이전트가 쉘 실행을 쉽게 할 수 있는 구조적 문제가 드러나면서 "장난감에서 프로덕션으로" 넘어가려면 보안이 필수라는 합의가 생겼습니다.

Hermes는 이 합의에 태생부터 맞춰져 있었습니다:

다층 승인 게이트
도구별·매개변수별 정책, allowlist/denylist
실행 격리
Docker, Singularity, Modal 샌드박스 — read-only root, cap-drop, pids_limit
Tirith 사전 실행 스캐너
쉘 명령을 실행 전에 분석하여 위험 패턴 차단
공급망 하드닝
v0.5.0 해시 핀닝, v0.8.0 OSV 스캐닝
시크릿 유출 차단
v0.7.0 outbound URL·base64·프롬프트 주입 경로 스캔
자격 증명 디렉토리 보호
~/.aws, ~/.ssh, ~/.docker 자동 차단
SSRF·타이밍 공격 방어
v0.8.0 보안 패스

"공개 인시던트가 보고된 적이 없다"는 것이 큰 차별점입니다. 기업 사용자는 이 조용함에 안도합니다.

Python 버전, uv, Node.js 같은 의존성을 인스톨러가 자동으로 처리합니다. "1분 안에 첫 대화"가 가능합니다.

TUI의 완성도 ​

  • 멀티라인 편집
  • 슬래시 명령 자동완성
  • 토큰 스트리밍 + 인터럽트
  • 세션 히스토리 탐색
  • 7개 내장 스킨

prompt-toolkit 기반으로 터미널 경험이 현대 IDE에 밀리지 않는 수준입니다. 이것이 사용자가 "또 하나의 CLI 에이전트"가 아니라 "쓸 만한 도구"로 느끼게 하는 결정적 요소입니다.

에러 메시지의 질 ​

에러가 날 때 무엇이 잘못되었고 어떻게 고칠지를 Markdown 포맷으로 친절히 알려줍니다. hermes doctor는 환경을 진단하고 문제를 짚어줍니다. 이것도 기능표에 안 나오지만 이탈률을 크게 낮춥니다.

마이그레이션 지원 ​

v0.2.0 출시 당시 이미 OpenClaw에서의 자동 마이그레이션(hermes claw migrate)을 제공했습니다. 경쟁자 사용자를 친절하게 받아들이는 태도가 이민율을 높였습니다.

원인 11 · "복사 가능한" 인기의 설계 ​

커뮤니티가 자발적으로 콘텐츠를 만들 템플릿을 Hermes는 의도적으로 제공했습니다.

바이럴 가능한 데모 포맷 ​

"Telegram에서 Claude Code를 제어하기"     → YouTube 튜토리얼
"Hermes에게 아침 브리핑 받기"             → Reddit 공유
"깃허브 이슈 자동 트리아지하기"             → 블로그 포스트
"Docker 배포를 자연어로"                  → Twitter 영상

각 시나리오가 5분 영상으로 만들 수 있는 길이이고, 즉각적 가치가 명확합니다. 크리에이터는 이런 아이디어를 찾기 어려운데, Hermes 공식 문서와 agentskills.io의 스킬 예제들이 콘텐츠 아이디어의 저수지 역할을 했습니다.

인플루언서의 재미 ​

Medium에 "Telegram에서 Claude Code 제어하기" 같은 글이 자발적으로 수십 편 올라온 것은 이 조합을 자랑하고 싶은 욕구를 자극했기 때문입니다. Hermes를 쓴다는 것 자체가 "최신 트렌드를 안다"는 신호가 되는 순간 바이럴이 시작됩니다.

원인 12 · OpenClaw와의 의도치 않은 시너지 ​

경쟁자로 시작했지만 보완재로 인식되면서 둘 다 이득을 봤습니다.

OpenClaw의 강점
오케스트레이션, 플러그인 폭, 대규모 배포 성숙도
Hermes의 강점
학습 루프, 장기 메모리, 자기개선
커뮤니티 합의
"Hermes가 OpenClaw를 대체하지 않는다. 둘은 다른 문제를 푼다"

이 프레이밍이 굳어지면서 사용자는 둘 다 배우는 것을 정당화할 수 있게 되었습니다. 경쟁 관계였다면 한쪽 선택이 다른 쪽 버림이었겠지만, 보완 관계로 인식되면 두 커뮤니티가 서로를 키웁니다.

OpenClaw의 345k 스타는 Hermes의 53k 스타를 누르는 대신 정당화합니다 — "OpenClaw가 이렇게 많이 쓰이는데, 그에 못지않은 Hermes도 써볼 가치가 있다"는 논리가 성립합니다.

원인 13 · 타이밍: AI 에이전트 하이프 사이클의 회복기 ​

Gartner의 하이프 사이클 관점에서 보면:

2023 ─ AutoGPT ── 혁신 촉발(Trigger)
    ↓
2024 ─ 과도한 기대(Peak of Inflated Expectations)
    "에이전트가 모든 일을 해줄 것"
    ↓
2024 말 ─ 환멸의 골짜기(Trough of Disillusionment)
    "에이전트는 아직 못 한다"
    ↓
2025 ─ 계몽의 경사(Slope of Enlightenment)
    "구체적으로 이런 작업은 가능하다"
    ↓
2026 ─ 생산성의 고원(Plateau of Productivity) 진입
    ✓ Hermes, Claude Code, OpenClaw 같은 "진짜 쓸 만한" 도구들

Hermes는 회복기의 첫 번째 우승자가 될 타이밍에 나왔습니다. 시장이 "뭐가 진짜 쓸 만한지" 다시 찾기 시작하는 순간, Nous Research의 브랜드 신뢰 + 명확한 차별점 + 오픈소스라는 조합이 잃을 게 없는 시도로 평가받았습니다.

인기가 이어지는 조건 ​

현재의 기세가 1년 이상 유지되려면 필요한 것들:

지속 가능성 체크리스트

  • v1.0 안정화 — API 브레이킹이 멈추면 엔터프라이즈 도입 가속
  • 엔터프라이즈 관리 콘솔 — SSO, RBAC, 감사 리포트, 중앙 정책
  • 문서화 격차 해소 — 현재 핵심 기능은 잘 설명되지만 운영 노하우가 부족
  • 스킬 마켓플레이스 수익화 — 기여자에게 인센티브 제공
  • 일본·유럽 커뮤니티 확장 — 현재 영어·중국어 중심
  • 보안 인시던트 제로 유지 — 한 번의 큰 사고가 신뢰를 갉아먹음
  • Nous Research의 조직 안정성 — 핵심 메인테이너 번아웃 방지

인기가 꺾일 수 있는 시나리오 ​

반대로 놓치면 안 되는 리스크:

주의해야 할 함정

  • 빅테크의 네이티브 에이전트 — Google Gemini Agent, OpenAI의 차세대 에이전트가 비슷한 수준으로 나올 가능성
  • 6주 릴리스 속도의 부작용 — API 안정성 희생, 기존 사용자 피로
  • 909 이슈가 2,000으로 — 트리아지 속도가 따라가지 못하면 신뢰 하락
  • 단일 조직 의존 — Nous Research에 문제가 생기면 전체 타격
  • 과도한 추상화 — 플러그인·스킬·MCP·프로파일이 새 사용자에게 혼란
  • 모델 제공자 정책 변경 — OpenRouter·Anthropic의 약관 변경 리스크
  • AutoGPT식 반짝 유행 — 6개월 후 잊히는 시나리오

결론: Hermes 인기는 구조적인가 일시적인가 ​

결론부터 말하면 구조적입니다. 지금 Hermes가 받고 있는 관심은 운이 아니라, 다섯 개의 독립적인 요인이 동시에 만나 만든 결과이며 그 중 서너 개는 쉽게 사라지지 않습니다.

구조적 요인 (쉽게 복제 불가):

  1. Nous Research 브랜드 신뢰
  2. 학습 루프라는 기술적 moat
  3. agentskills.io 네트워크 효과
  4. 오픈소스 + MIT의 경제적 해자
  5. 6주 7릴리스가 쌓은 커뮤니티 기대

일시적 요인 (경쟁자가 따라잡을 수 있음):

  1. 시장 타이밍(빅테크 에이전트 출시로 상쇄 가능)
  2. OpenClaw와의 보완재 프레이밍(경쟁이 격화되면 변함)
  3. 연구자·해커 커뮤니티의 초기 실험 열풍

그래서 1–2년 관점에서는 Hermes가 안정적으로 성장할 가능성이 높고, 3–5년 관점에서는 학습 루프와 스킬 생태계의 성숙도에 달렸습니다. 지금이 채택을 고민할 가장 좋은 타이밍이라는 판단은 이 분석에서 나옵니다.
